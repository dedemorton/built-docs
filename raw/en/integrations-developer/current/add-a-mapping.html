Edit field mappingsedit

Ingest pipelines create fields in an Elasticsearch index, but don’t define the fields themselves. Each field needs a defined data type, or mapping.

Mapping is the process of defining how a document, and the fields it contains, are stored and indexed. Each document is a collection of fields, which each have their own data type. When mapping your data, you create a mapping definition, which contains a list of fields that are pertinent to the document. A mapping definition also includes metadata fields, like the _source field, which customize how a documentâ€™s associated metadata is handled.

To learn more, see mapping.
